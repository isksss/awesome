# レビュー方針

このリポジトリの資料は、読みやすさと正確さを保つために半年ごとに見直します。

完璧な資料を一度で作るより、古くなったところを見つけて少しずつ直す方針です。

## 対象

- Spring / Spring Security / Spring MVC
- MySQL / PostgreSQL
- Nginx
- TLS / HTTP
- 重要・危険・バージョン依存の記述
- 外部出典リンク

## 方針

- 公開できない実システム固有情報は含めない。
- 出典が古くなった記述は更新または削除する。
- 悪用可能な攻撃再現手順は追加しない。
- 架空ログ、架空構成、無害なサンプルだけを使う。
- 読んだ人が次の行動を取りやすいように、具体例や確認コマンドを添える。
