# AGENTS.md

このリポジトリで AI エージェントが作業するときのルールです。

## 基本方針

- 日本語で簡潔に回答する。
- 事実、既存規約、最小変更を優先する。
- 公開リポジトリであることを前提に、秘密情報や実システム固有情報を入れない。
- 既存のディレクトリ構成、命名、文体、検証コマンドに合わせる。
- 無関係な整形、リファクタ、依存追加を避ける。
- ユーザーの未 commit 変更を勝手に戻さない。

## リポジトリの目的

このリポジトリは、汎用的な技術資料、学習メモ、運用メモを置く場所です。

- `docs/fundamentals/`: Git、Docker、Linux コマンドなどの基礎資料
- `docs/process/`: レビュー方針などの運用メモ
- `docs/`: Nuxt サイトで本文とスライドを動的表示する Markdown 教材
- `scripts/`: 検証、生成、ビルド用スクリプト
- `app/`: Nuxt + Nuxt UI の静的サイト

## 書き方

- 初学者にも読める説明から始め、実務での注意点まで書く。
- コマンドは、目的、例、注意点をセットで書く。
- 削除、上書き、権限変更、force 操作など危険な操作は、確認コマンドや影響範囲も書く。
- バージョン依存や環境差分がある場合は、確認方法を書く。
- 実在組織、実環境のログ、認証情報、内部設定値は使わない。
- サンプルは架空値を使う。

## 変更時の確認

作業前:

```sh
git status --short
```

Markdown やリンクを変更した場合:

```sh
pnpm check:links
```

`docs/` 配下の教材を変更した場合:

```sh
pnpm check:lessons
```

全体を確認する場合:

```sh
pnpm build
```

Node と pnpm は mise で管理します。作業前に `mise install` で環境をそろえます。

## 作業カテゴリ

必要に応じて、以下の作業モードを使います。

- `research`: 調査、原因分析、影響範囲確認
- `implement`: 合意済み方針の実装
- `review`: 差分確認、回帰や検証不足の指摘
- `branch`: 作業ブランチ作成
- `commit`: 差分確認後の commit
- `merge-request`: PR / MR 作成

## 禁止事項

- 秘密情報、個人情報、実環境設定値を追加しない。
- 悪用可能な攻撃再現手順を詳細化しない。
- `git reset --hard` や `git checkout --` でユーザー変更を戻さない。
- 生成物や依存関係を、必要性なしに追加しない。
- すべてのコンテンツは `docs/` 配下に配置する。
- `docs/` の教材検査を壊す再配置をしない。

## 完了報告

完了時は以下を簡潔に報告する。

- 変更内容
- 実行した検証
- 未実行の検証があれば理由
- 残リスクがあれば内容
